let 和 const 命令

  1. let 声明的变量只在它所在的代码块有效

  1. for 循环设置循环变量的那部分与循环体内部的作用域是独立的。

  2. var 声明的变量会发生“变量提升”现象，即变量可以在声明之前使用，此时变量的值为 undefined. let 命令要求变量一定要在声明之后才可以使用。

  3. 在块级作用域中，只要存在 let 命令，其声明的变量就“绑定”该区域，不会受外部影响:

    在 ES6 中，如果区块中存在 let 或者 const 的话，该区块内使用这些命令声明的变量就形成了封闭作用域，称为暂时性死区(temporal dead zone – TDZ)

  4. 在使用 let 声明变量前，如果对该变量使用 typeof 操作符的话，获取到的值会是 ReferenceError. 如果根本没声明该变量的话，获取到的是 "undefiend"

  5. 在相同作用域内，不允许使用 let 重复声明同一个变量
  6. ES5 中没有块级作用域的定义，而 ES6 提供的 let 实际上为 JavaScript 添加了块级作用域

  7. ES6 对在块级作用域中声明函数有以下规定:

    • 允许在块级作用域内声明函数
    • 函数声明类似 var, 会提升到全局作用于或者是函数作用域头部
    • 函数声明还是提升到所在块级作用域的头部

由于 ES5 对于在块级作用域中声明函数的执行方式与 ES6 不同，所以不建议在块级作用域中声明函数，如果要使用此方式的话，建议使用函数表达式来声明函数，而不是函数声明。

  1. const 声明的是一个只读的常量，其值不能改变。且一旦声明了函数，则必须立即初始化。const 的作用域与 let 相同，都是只在声明的块级作用域内生效

  2. const 实际上保证的是变量指向的那个内存地址不得改动。对于基本数据类型，比方说数值，字符串等，值就保存在变量指向的内存地址上，等同于常量。但对于对象和数组等来说，变量指向的地址只是一个指针， const 保证该指针是固定的，但是指针指向的数据结构就不是 const 所能保证的。

  1. 如果想将对象冻结的话，使用 Object.freeze 方法（冻结对象无法添加新属性）。如果除了将对象冻结以外还想冻结对象的属性的话:

  2. var 和 function 命令声明的全局变量，为顶层对象的属性（在浏览器中是 window, 在 Node 中是 global）. let, const 和 class 创建的全局变量不属于顶层对象

  3. 如果函数不是作为对象的方法运行的话，而是作为函数运行，函数中的 this 会返回顶层对象

变量的解构赋值

  1. 解构不成功的话，变量的值会等于 undefined
  2. ES6 内部会使用严格相等运算符来判断数组的某个位置是否有值，当数组成员严格等于 undefined 时，解构赋值所设置的指定默认值才会生效（null 等于 undefined, 但是不严格相等）

  3. 当解构模式是嵌套对象，且子对象所在的父属性不存在时，将会报错:

  4. 类数组对象中都有一个 length 属性。对字符串进行解构赋值的话，字符串会被换成一个类数组的对象

字符串的扩展

  1. JavaScript 内部，字符以 UTF-16 的格式存储，对于需要4个字节存储的字符，即 Unicode 码点大于 0xFFFF 的字符来说，JavaScript 会认为它们是两个字符

    从上可知，charAt 和 charCodeAt 方法无法正确处理4个字节的字符。故ES6 提供了 codePointAt 方法来处理4个字节存储的字符，返回该字符的码点。

  2. for...of 循环会正确识别 32 位的 UTF-16 字符，可以识别大于 0xFFFF 的码点。传统 for 语句无法识别这样的码点

  3. 通过 normalize() 方法可以将字符的不同表示方法统一为同样的形式，称为 Unicode 正规化
  4. 可以通过 includes() 来表示是否找到了参数字符串，startsWith() 来判断参数字符串是否在原字符串的头部， endsWith() 判断参数字符串是否在原字符串的尾部。三个方法都可以添加第二个参数来指定搜索的位置。includes() 和 startsWith 的第二个参数表示从第 n 个位置到字符串结束，而 endsWith 针对的是前 n 个字符

  5. repeat 方法返回新的字符串，表示将原字符串重复 n 次。当 n 为小数时，会执行向下取整；当 n 为 0 或者 NaN 时，返回空字符串；当 n 为 Infinity 或者负数时，会报错

  6. padStart 和 padEnd 分别用于头部和尾部补全，第一个参数为整个字符串的长度，第二个参数为填充的字符。当补全的字符串长度小于原字符串长度时，返回原字符串

正则的扩展

  1. 字符串对象有4个方法可以使用正则表达式: match(), replace(), search() 和 split(). ES6 在这4个方法内部全部调用 RegExp 的实例方法，即字符串中与正则相关的方法都是定义在 RegExp 对象上:

    • String.prototype.match 调用 RegExp.prototype[Symbol.match]
    • String.prototype.replace 调用 RegExp.prototype[Symbol.replace]
    • String.prototype.search 调用 RegExp.prototype[Symbol.search]
    • String.prototype.split 调用 RegExp.prototype[Symbol.split]

  2. y 修饰符与 g 修饰符相似，也是全局匹配，但是 g 修饰符只要剩余位置中存在匹配即可，y 修饰符确保匹配必须从剩余的第一个位置开始

  3. source 属性返回正则表达式的正文，flags 属性返回正则表达式的修饰符

    先行断言：x 只有在 y 前面才匹配，必须写成 /x(?=y)/.

    先行否定断言是指 x 只有不在 y 前面才匹配，必须写成 /x(?!y)/ 。

    后行断言指的是 x 只有在 y 后面才匹配，必须写成 /(?<=y)x/.

    后行否定断言指 x 只有不在 y 后面才匹配，必须写成 /(?<!y)x/ 。

  4. 具名组匹配( Named Capture Groups): 给每一个组匹配指定名字:

    每个组名都是通过 ?<name> 的方式进行命名，然后在结果的 groups 属性上引用该组名

  5. 使用 replace() 方法替换时，使用 $<name> 引用具名组。在正则表达式内部引用某个具名组的话，使用 \k<name>的写法
  6. 使用 ... 运算符和 Array.from() 方法可以将遍历器转换为数组

数值的扩展

  1. 二进制和八进制分别使用 0b/0B 和 0o/0O 表示。通过 Number 方法，可以将带前缀的字符串转换为十进制:

  2. Number 添加了两个方法: Number.isFinite 和 Number.isNaN 用于判断参数是否为有限值和 NaN.当参数为非数值时，一律返回 false。另外，这两个方法与传统的 isFinite() 和 isNaN() 的区别在于，传统的全局方法会将非数值的转为数值以后再进行判断。

  3. parseInt() 和 parseFloat() 从全局方法移植到了 Number 对象上

  4. JavaScript 内部，整数和浮点数采用同样的存储方法，所以1和1.0被视为同一个值
  5. 可以通过 + 等符号将一个值转为数值

函数的扩展

  1. 对函数使用参数默认值时，不允许有同名的参数
  2. 注意对象解构赋值与函数参数默认值的区别
  3. 当函数参数设置了默认值时，传入 undefined 将会触发默认值
  4. 指定了默认值后，函数的 length 属性将会返回没有指定默认值的参数的个数
  5. 函数的参数设置默认值后，会形成一个单独的作用域
  6. rest 参数: ...变量名 可以用于替代 arguments 对象。arguments 是一个类数组对象，必须使用 Array.prototype.slice.call 将其转换为数组。而 rest 参数本身就是一个数组

  8. 如果箭头函数直接返回一个对象，必须在对象外面加上括号，因为大括号会被解释为代码块

  9. 箭头函数的 this 对象是指定义时所在的对象，不是使用时所在的对象

通过扩展运算符可以替代 `apply` 和 `push`

    JS 中，直接复制数组的话，仅仅复制了指向底层数据结构的指针，而不是复制一个全新的数组。ES5 中通过 concat() 来复制一个数组，而 ES6 可以使用扩展运算符来复制

  1. 合并数组

  2. 将字符串转为数组

  4. Array.of() 用于将一组值转换为数组

与数组构造函数的差别在于参数个数不同时会有差异:

  1. 数组实例的 copyWithin 方法会在当前数组内部将指定成员复制到其他位置，覆盖原有的成员，然后返回当前数组

  2. 数组实例的 find 方法用于找出第一个符合条件的数组成员，没有符合条件的话，返回 undefined 。findIndex方法用于查找符合条件的第一个数组成员的位置，如果所有成员都不符合条件时，返回 -1

对象的扩展

  1. Object.assign() 方法用于两个对象的合并，将源对象的所有可枚举属性，复制到目标对象上

  2. Object.assign() 只拷贝源对象本身的属性（不拷贝继承属性），不拷贝不可枚举的属性

  3. Object.assign() 实现的是浅拷贝
